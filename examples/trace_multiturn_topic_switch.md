# 예시 11: 멀티턴 주제 전환 — 맥락 독립성

> **학습 목표**: 2턴에서 완전히 다른 주제로 전환될 때,
> FM이 이전 대화 이력에 **끌려가지 않고** 새 주제를 독립적으로 처리하는 패턴을 추적한다.

---

## 대화 흐름

```
Turn 1: "오늘 소나타 생산 현황"
Turn 2: "심야 교대 불량이 좀 심한 것 같은데 확인해줘"  ← 완전히 다른 주제
```

---

## Turn 1: 오늘 소나타 생산 현황

### IntentAgent → InfoAgent → Tool
- intent: `production_query`, model=SONATA, period=today
- Tool: `get_production_summary({"period": "today"})`

### 응답
```
## 오늘 소나타 생산: 347/360대 (달성률 96.4%)
```

---

## Turn 2: 심야 교대 불량이 좀 심한 것 같은데 확인해줘

### IntentAgent

#### 🔷 FM 입력 (→ Gemini gemini-2.0-flash)
- **Human** (`_build_context` 적용):
```
[이전 대화 이력]
사용자: 오늘 소나타 생산 현황
의도: production_query
응답 요약: 오늘 소나타 생산: 347/360대 (달성률 96.4%)

[현재 질문]
심야 교대 불량이 좀 심한 것 같은데 확인해줘

위 대화 이력을 참고하여 현재 질문의 의도를 분석하세요.
```

#### 🔶 FM 출력 (← Gemini)
```json
{
  "intent": "defect_query",
  "detail": {
    "line": "",
    "model": "",
    "shift": "MIDNIGHT",
    "date_from": "",
    "date_to": "",
    "period": "",
    "keyword": ""
  },
  "reasoning": "심야 교대의 불량 현황 확인 요청. 이전 대화(소나타 생산)와는 별개의 새 주제."
}
```

> **핵심 — 맥락 독립**: FM이 이전 대화(소나타)에 끌려가지 않음.
> - model은 빈 값 (소나타로 설정하지 않음)
> - shift=MIDNIGHT만 추출
> - reasoning에 "이전 대화와는 별개의 새 주제"라고 명시

### InfoAgent → Tool
- `get_defect_stats({"shift": "MIDNIGHT"})`

> **참고**: `get_defect_stats`는 shift 파라미터를 직접 지원하지 않음.
> 실제로는 FM이 전체 불량 데이터를 가져온 뒤 심야 교대분만 필터링하여 응답하거나,
> `get_shift_analysis`를 대신 호출할 수 있음.

### 응답
```
## 심야 교대(MIDNIGHT) 불량 현황

| 라인 | 불량 건수 | 불량률 |
|------|----------|--------|
| LINE-1 | 45건 | ⚠️ 2.12% |
| LINE-2 | 52건 | ⚠️ 3.85% |
| LINE-3 | 38건 | ⚠️ 2.78% |

심야 교대 전체 불량률 **2.06%**로 주간(1.27%), 야간(1.64%) 대비 가장 높습니다.
LINE-2 심야의 불량률(3.85%)이 특히 높아 주의가 필요합니다.
```

---

## FM 호출 요약

| Turn | 질문 | intent | 이전 이력 활용 |
|------|------|--------|--------------|
| 1 | "오늘 소나타 생산 현황" | production_query | 없음 |
| 2 | "심야 교대 불량이..." | defect_query | 이력 전달되나 새 주제 독립 처리 |

---

## 학습 포인트

1. **맥락 독립성**: 이전 대화 이력이 FM에 전달되지만, 새 질문의 주제가 다르면 FM이 독립적으로 판단. "소나타" 문맥에 오염되지 않음.
2. **reasoning의 역할**: FM이 "이전 대화와 별개"라고 스스로 판단을 명시함으로써 의도분류의 정확성 향상.
3. **대명사 없음**: "심야 교대 불량"은 대명사가 없고 새 주제를 직접 명시. `_build_context()`가 이력을 주입해도 FM은 새 질문에 집중.
4. **주간/야간 비교 자동 포함**: FM이 심야만 물어봤는데 다른 교대와의 비교를 자동 추가하여 맥락 보강.
