# 예시 12: 멀티턴 대명사 해소 — "그럼 ~은?" 패턴

> **학습 목표**: "그럼", "거기서", "그 라인" 등 대명사가 사용될 때
> `_build_context()`와 FM의 협력으로 이전 대화의 엔티티를 정확히 참조하는 과정을 추적한다.

---

## 대화 흐름

```
Turn 1: "아이오닉6 이번 달 생산 추이 보여줘"
Turn 2: "그럼 불량 추이는?"                    ← "그럼" = 아이오닉6 유지
Turn 3: "그 라인 정지 이력도 알려줘"            ← "그 라인" = LINE-3 (아이오닉6의 라인)
```

---

## Turn 1: 아이오닉6 이번 달 생산 추이 보여줘

### IntentAgent
- 🔶 FM 출력: `trend_analysis`, model=IONIQ6, period=this_month

### InfoAgent → Tool
- `get_production_trend({"model": "IONIQ6", "days": 28})`

### 응답
```
## 아이오닉6 2월 생산 추이 (LINE-3)
| 주차 | 평균 달성률 | 평균 불량률 |
|------|-----------|-----------|
| 1주 | 94.2% | 2.1% |
| 2주 | 93.0% | 2.5% |
| 3주 | 91.8% | ⚠️ 3.2% |
| 4주 | 93.5% | 2.7% |
```

### 이력 저장
```python
conversation_history = [
    {"user": "아이오닉6 이번 달 생산 추이 보여줘", "intent": "trend_analysis",
     "answer": "아이오닉6 2월 생산 추이 (LINE-3)...1주 94.2%...3주 91.8%..."}
]
```

## Turn 2: 그럼 불량 추이는?

### IntentAgent

#### 🔷 FM 입력 (→ Gemini gemini-2.0-flash)
```
[이전 대화 이력]
사용자: 아이오닉6 이번 달 생산 추이 보여줘
의도: trend_analysis
응답 요약: 아이오닉6 2월 생산 추이 (LINE-3)...

[현재 질문]
그럼 불량 추이는?

위 대화 이력을 참고하여 현재 질문의 의도를 분석하세요.
'그럼', '거기', '그 라인' 등 대명사는 이전 대화에서 언급된 대상을 참조합니다.
```

#### 🔶 FM 출력 (← Gemini)
```json
{
  "intent": "defect_query",
  "detail": {
    "line": "",
    "model": "IONIQ6",
    "shift": "",
    "date_from": "2026-02-01",
    "date_to": "2026-02-28",
    "period": "this_month",
    "keyword": "추이"
  },
  "reasoning": "'그럼'은 이전 대화의 아이오닉6를 유지하면서 관점을 '생산 추이'에서 '불량 추이'로 전환"
}
```

> **핵심 — 대명사 해소 (엔티티 유지)**:
> - "그럼" → 이전 대화의 **아이오닉6** 엔티티를 유지
> - "불량 추이" → 의도를 `defect_query`로 전환
> - model=IONIQ6가 이전 턴에서 **자동 이월**

### InfoAgent → Tool
- `get_defect_stats({"model": "IONIQ6", "date_from": "2026-02-01", "date_to": "2026-02-28"})`

### 응답
```
## 아이오닉6 2월 불량 현황
| 유형 | 건수 |
|------|------|
| 전장(electric) | 62건 (44%) |
| 조립(assembly) | 45건 (32%) |
| 용접(welding) | 22건 (15%) |
| 도장(paint) | 13건 (9%) |

⚠️ EV 특성상 전장 불량(44%)이 가장 높습니다.
```

## Turn 3: 그 라인 정지 이력도 알려줘

### IntentAgent

#### 🔷 FM 입력 (→ Gemini gemini-2.0-flash)
```
[이전 대화 이력]
사용자: 아이오닉6 이번 달 생산 추이 보여줘
의도: trend_analysis
응답 요약: 아이오닉6 2월 생산 추이 (LINE-3)...

사용자: 그럼 불량 추이는?
의도: defect_query
응답 요약: 아이오닉6 2월 불량 현황...전장 62건(44%)...

[현재 질문]
그 라인 정지 이력도 알려줘
```

#### 🔶 FM 출력 (← Gemini)
```json
{
  "intent": "downtime_query",
  "detail": {
    "line": "LINE-3",
    "model": "",
    "shift": "",
    "date_from": "",
    "date_to": "",
    "period": "",
    "keyword": ""
  },
  "reasoning": "'그 라인'은 이전 대화에서 아이오닉6가 생산되는 LINE-3을 참조. 정지 이력 요청이므로 downtime_query."
}
```

> **핵심 — 간접 참조 해소**:
> - "그 라인" → 아이오닉6 → LINE-3 (**2단계 추론**)
> - 사용자가 "LINE-3"을 직접 말한 적 없음
> - FM이 이력의 "아이오닉6 → LINE-3" 매핑을 이해하고 해소

### InfoAgent → Tool
- `get_downtime_history({"line": "LINE-3"})`

### 응답
```
## LINE-3 (EV라인) 정지 이력
| 일시 | 사유 | 시간 | 설명 |
|------|------|------|------|
| 2/24 22:00 | 설비 고장 | 95분 | 배터리 모듈 조립기 이상 |
| 2/18 14:00 | 자재 부족 | 60분 | 배터리 셀 공급 지연 |
| 2/10 06:00 | 정기 정비 | 180분 | EV 라인 월간 정비 |

EV 라인 특성상 배터리 관련 정지(조립기 이상, 셀 공급 지연)가 주요 원인입니다.
```

---

## FM 호출 요약 (3턴 합계)

| Turn | 질문 | 대명사 | 해소 결과 |
|------|------|--------|----------|
| 1 | "아이오닉6 이번 달 생산 추이" | 없음 | — |
| 2 | "**그럼** 불량 추이는?" | "그럼" | → 아이오닉6 유지 |
| 3 | "**그 라인** 정지 이력도" | "그 라인" | → 아이오닉6 → LINE-3 (2단계) |

**총 FM 호출: 9회** (3턴, 각 3회)

---

## 학습 포인트

1. **엔티티 이월**: "그럼"이 이전 턴의 IONIQ6를 자동으로 유지. `_build_context()`가 이력을 주입하여 FM이 판단.
2. **2단계 간접 참조**: "그 라인" → 아이오닉6 → LINE-3. 사용자가 명시하지 않은 라인 ID를 FM이 추론으로 도출.
3. **의도 전환과 엔티티 유지의 분리**: Turn 2에서 의도는 trend_analysis → defect_query로 바뀌지만 대상(IONIQ6)은 유지. 의도와 엔티티를 독립적으로 처리.
4. **_build_context()의 역할**: 최근 5턴의 이력을 구조화된 형태로 FM에 전달. 이것이 없으면 "그럼", "그 라인"을 해소할 수 없음.
5. **대명사 해소 vs 주제 전환**: 예시 11(주제 전환)과 대조적. 대명사가 있으면 이전 엔티티 참조, 없으면 새 주제 독립 처리.
